# ページ構成

ポートフォリオサイト（my-portfolio2026）のページ構成と、各セクションに載せる内容をまとめたドキュメント。

- 関連 Issue: [#1 ページの構成を決める](https://github.com/kainholy/my-portfolio2026/issues/1)

---

## 1. 全体方針

- **単一ページ（1ページ完結）構成**とする。
- 各コンテンツはページ内アンカーで遷移する。

---

## 2. ページ構成

- ヘッダー / ナビ
- ファーストビュー
- WORKS（実績）
- ABOUT（自己紹介）
- CAN DO（できること）
- CONTACT
- フッター

---

## 3. セクション別 詳細要件

### 3.1 ヘッダー / ナビ

- ロゴ（名前 or 屋号）
- ナビゲーションリンク：WORKS / ABOUT / CAN DO / CONTACT

### 3.2 ファーストビュー

- キャッチコピー（簡潔な一言）
- 肩書き / 一言の自己紹介
- スクロール誘導（任意）

### 3.3 WORKS（実績）

WORKS は **詳細ページを実装** する予定。トップページには一覧（サマリー）を表示し、各案件の詳細は詳細ページに表示する。

#### トップページ（一覧表示） ※実績1件ごとに繰り返し

- 案件名
- サムネイル画像（スクリーンショット）
- 概要（どんなサイト・案件か）
- 使用技術
- 詳細ページへのリンク

#### 詳細ページ（1案件ごと）

- 案件名 / クライアント名（掲載可の範囲で）
- サムネイル画像（スクリーンショット）
- 概要（どんなサイト・案件か）
- 担当した役割
- 使用技術
- 取り組んだ内容 / 工夫した点
- 成果（あれば）
- サイトURL（公開済みかつ掲載可の場合）

### 3.4 ABOUT（自己紹介）

- 氏名
- 経歴（職歴・現在の立場）
- 得意領域 / 専門性
- 人柄・仕事への姿勢（一言）
- 顔写真 or アバター（任意）

### 3.5 CAN DO（できること）

- 提供できるサービス（例：サイト制作、リニューアル、保守・更新）
- 対応できる技術・スキル
- 対応範囲（どこからどこまで任せられるか）

### 3.6 CONTACT

- 連絡手段（メールアドレス / フォーム）
- 問い合わせへの一言
- SNS / 外部リンク（任意）

### 3.7 フッター

- コピーライト
- 外部リンク（GitHub / X / Zenn 等）
- ナビ再掲（任意）
